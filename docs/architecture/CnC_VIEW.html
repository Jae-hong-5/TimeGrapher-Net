TimeGrapherNet — Component-and-Connector View

Notation: UML Component Diagram · runtime components & typed connectors across three concurrent threads

이 다이어그램은 C&C 뷰를 UML 컴포넌트 다이어그램으로 표현한다. «component» 박스는 런타임 인스턴스, 롤리팝/소켓은 제공/요구 인터페이스, «thread» 패키지(폴더 모양)는 동시 실행 스레드, 점선 화살표는 «event»·«async-dispatch»·«binding» 의존, 실선 화살표는 «call-return» 호출이다. 원통은 «shared-data» 저장소다. 같은 내용의 Markdown 본은 CnC_VIEW.md / CnC_VIEW_EN.md.

UML Component Diagram

Notation Legend (UML → C&C)

call-return — synchronous call (solid) event — async notify (dashed) shared-data — buffer access async-dispatch — thread marshal publish-subscribe — fan-out binding — UI data bind

UML elements used: «component» box with the two-tab component icon · lollipop ○ = provided interface · socket ⊃ = required interface (lollipop+socket joined = assembly connector) · «thread» package (folder shape) = a concurrent thread · cylinder = «shared-data» store · dashed box = «socket» extension point (runtime-injected). Each thread boundary is crossed only by the dashed event and async-dispatch connectors plus the shared-data store.

Constraints & Rationale

Core components do not call UI/platform componentsAnalysisWorker/pipeline only publish AnalysisFrameReady and know no consumer — Core no-dependency principle (Module Uses View)
Thread boundaries crossed only by event + shared-data + async-dispatchNo cross-thread direct method call; UI updates always via Dispatcher.UIThread.Post
