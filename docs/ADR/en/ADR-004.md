# ADR 4: Separate App, Test, and Verify Module Structure for AI Usage, TDD Support, and Team Collaboration

The TimeGrapher system requires a high level of modifiability and extensibility so that new measurement metrics or graph analysis modes can be added in the future. Because the current team has six members working on the same codebase at the same time, the structure must support parallel development and minimize code conflicts. In addition, to ensure the correctness of code generated by on-device AI integration and AI coding assistants, the project needs a TDD environment that automatically verifies unit behavior and careful human code review. The team must also share these architectural decisions to reduce communication waste.

## Decision

- **We will separate the fundamental project structure into independent `App`, `test`, and `verify` modules.**
- **We will adopt Test-Driven Development (TDD) practices to ensure that the core domain logic remains highly testable and UI-agnostic.**
- **We will enforce automated testing via the `test` and `verify` modules as a mandatory step before any code commit, providing a verified baseline for human review of AI-assisted code.**

## Rationale

- **Collaboration efficiency**: Because the core domain, UI, and tests are physically separated into different modules, six team members can work at the same time with significantly fewer merge conflicts and more independent parallel work.
- **Safety net for AI code**: Code suggested or generated by AI can contain unexpected errors. Tests written through TDD serve as the first safety net, and when the six team members review each other's code, test results provide a more objective and efficient basis for merging.
- **Rejected alternative**: Placing all code inside a single `App` module was also considered. However, this approach was rejected because team members would have to understand and manually verify large amounts of AI-generated code, creating an excessive code review burden.

## Status

Accepted

## Consequences

Positive:

- Minimal test automation greatly reduces the review burden for AI-generated code and improves the project's overall development speed.
- An automated test foundation provides the reliability needed to merge code written by team members or AI into the main branch with confidence.

Negative:

- Because the team must follow TDD principles, the learning curve and development time for initial feature implementation may increase for all team members.
- The team must manage dependencies among the three modules, and additional maintenance cost is required to keep the CI pipeline and ADR documents up to date as work from all six members is integrated.
